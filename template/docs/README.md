# Documentation Index

Use `docs/` for durable project documentation that should survive chat history resets.

Recommended initial docs:

- architecture overview
- domain model
- integration contracts
- local development bootstrap
- testing strategy
- naming conventions
- decision records in `docs/adr/`
- workflow expectations and validation norms if they differ from the template defaults

Rule:

- if structure, architecture, interfaces, or workflow change, update docs in the same PR
